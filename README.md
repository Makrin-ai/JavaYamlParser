# Домашняя работа: Парсер учебного конфигурационного языка

1. ОБЩЕЕ ОПИСАНИЕ

Данный проект реализует инструмент командной строки для обработки учебного конфигурационного языка (УКЯ).  
Проект предназначен для трансляции текстовых конфигураций в формат YAML с поддержкой вложенных словарей и констант.  

Особенности проекта:  
- Поддержка однострочных комментариев (`C это комментарий`).  
- Обработка чисел (целых и дробных) и логических значений (`true/false`).  
- Обработка строк в кавычках и простых строк без кавычек.  
- Поддержка вложенных словарей произвольной глубины.  
- Объявление констант (`имя is значение`) и ссылки на них (`[имя]`).  
- Выявление синтаксических ошибок с подробными сообщениями.  
- Форматированный вывод в YAML с использованием библиотеки [SnakeYAML](https://bitbucket.org/asomov/snakeyaml).  

Проект написан на Java и не использует внешние скрипты для парсинга, однако применяется специализированная библиотека SnakeYAML для форматирования вывода. 

Тестовые данные находятся в папке lib/tests
Код на языке Java находится в папке src -> Main.java

2. ОПИСАНИЕ ФУНКЦИЙ И НАСТРОЕК

Основные функции

1). Чтение входного файла
   - Считывает конфигурацию из текстового файла, указанного пользователем.  
   - Игнорирует пустые строки и комментарии.  

2). Парсинг констант
   - Обнаруживает объявления вида `имя is значение`.  
   - Поддержка строк, чисел, логических значений и словарей.  
   - Константы могут использоваться в последующих словарях через `[имя]`.  

3). Парсинг словарей
   - Поддержка вложенных словарей `{ключ = значение, ...}`.  
   - Автоматическое распознавание вложенных конструкций и констант.  
   - Проверка синтаксиса и генерация исключений при ошибках.  

4). Форматирование YAML
   - Генерация читаемого YAML с отступами и блоковым стилем.  
   - Клонирование словарей для предотвращения дублирования ссылок в YAML.  

Настройки

- Конфигурация YAML через `DumperOptions`:
  - Блочный стиль (`BLOCK`)  
  - Отступ 2 пробела  
  - Pretty flow для улучшенной читаемости

3. КОМАНДЫ ДЛЯ СБОРКИ И ЗАПУСКА

Сборка проекта

Используется стандартный Java-компилятор:

```bash
javac -cp lib/snakeyaml-2.0.jar -d out/production/MyYamlProject src/Main.java
----

Запуск проекта
java -cp "out/production/MyYamlProject:lib/snakeyaml-2.0.jar" Main
После запуска программа запросит путь к файлу с конфигурацией:
Введите путь к файлу с тестом:
Пример ввода польователем:
/Users/user/Downloads/MyYamlProject/lib/tests/course.txt
или
/Users/user/Downloads/MyYamlProject/lib/tests/web_server.txt
4. ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ
Пример 1: Конфигурация курса
Файл course.txt:
course_name is "Java Programming"
students_count is 25
topics is {
    t1 = "Основы"
    t2 = "ООП"
    t3 = "Алгоритмы"
}

course is {
    name = [course_name]
    students = [students_count]
    syllabus = [topics]
}
Эталонный вывод программы:
topics:
  t1: Основы
  t2: ООП
  t3: Алгоритмы
course:
  name: Java Programming
  students: 25
  syllabus:
    t1: Основы
    t2: ООП
    t3: Алгоритмы
